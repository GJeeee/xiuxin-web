# 飞书梁派命理机器人

在飞书里像聊天一样，按**梁湘润《子平基础概要》**体系问命理。

> 适合零基础：只需完成一次飞书配置 + 填 API Key，之后**双击启动**即可。

---

## 你需要准备（3 样东西）

| 项目 | 说明 | 怎么获取 |
|------|------|----------|
| **飞书账号** | 企业版或个人版均可 | 已有即可 |
| **飞书应用** | 机器人身份 | 下面「第一步」免费创建 |
| **DeepSeek API Key** | 驱动 AI 回答 | [platform.deepseek.com](https://platform.deepseek.com) 注册后创建 |

DeepSeek 费用很低（一般几块钱能用很久）。

## 第一步：创建飞书机器人（只需做一次，约 15 分钟）

### 1. 打开飞书开放平台

浏览器访问：**https://open.feishu.cn/app**

登录你的飞书账号 → 点击 **「创建企业自建应用」**

- 应用名称：`梁派命理`（随意）
- 应用描述：命理学习助手

### 2. 获取 App ID 和 App Secret

进入应用 → 左侧 **「凭证与基础信息」**

复制保存：

- **App ID**
- **App Secret**（点击显示）

### 3. 启用机器人

左侧 **「应用能力」→「机器人」** → 开启

### 4. 配置权限

左侧 **「开发配置」→「权限管理」** → 搜索并开通：

- `im:message`
- `im:message:send_as_bot`
- `im:message.group_at_msg`（若在群里用）
- `im:chat:readonly`（可选）

点击 **「批量开通」** 或逐个申请（测试阶段通常自动通过）。

### 5. 配置事件订阅（重要 · 群聊不回 90% 在这里）

左侧 **「开发配置」→「事件与回调」**

1. 订阅方式：选 **「使用长连接接收事件（WebSocket）」**  
   ⚠️ **不要选「请求地址 URL」**
2. **先在本机运行** `./启动机器人.command`，等到终端出现：
   ```text
   [Lark] ... connected to wss://msg-frontier.feishu.cn ...
   ```
3. **保持机器人窗口开着**，回到飞书开放平台，点击 **「添加事件」**：
   - `im.message.receive_v1`（接收消息）
4. 点击 **保存**。此时页面应显示长连接 **「已连接」**
5. 若保存时显示「未检测到连接」→ 说明机器人没在跑，或 SSL 未修复

**群聊权限（必开）：**

在 **权限管理** 搜索并开通：

- `im:message`
- `im:message:send_as_bot`
- `im:message.group_at_msg`（**接收群聊 @ 机器人消息，缺这个群聊不回**）

### 6. 发布应用

左侧 **「应用发布」→「版本管理与发布」** → 创建版本 → 提交发布

（个人测试可用「仅自用」或邀请自己为测试用户）

### 7. 把机器人拉进聊天

- **单聊**：飞书搜索你的应用名 → 打开 → 发消息
- **群聊**：群设置 → 添加机器人 → 选择「梁派命理」

## 常见问题

### Q：建了群 @ 机器人，但不回复？

按顺序排查：

**① 终端窗口开着吗？**  
必须运行 `./启动机器人.command`，且看到 `[Lark] connected`。

**② 发 @ 后，终端有没有这两行？**
```text
事件到达 message_id=...
收到消息 chat=... text=帮助
```
- **完全没有** → 飞书事件没推过来（见下方 ③）
- **有「事件到达」但没回复** → 把终端报错发给我
- **有「send failed」** → 权限不足，补开 `im:message:send_as_bot`

**③ 重做事件订阅（最常见原因）**

1. 先运行 `./排查飞书事件.sh` 或 `./启动机器人.command`
2. 看到 `connected` 后，打开 https://open.feishu.cn/app → 你的应用
3. **事件与回调** → 确认是 **WebSocket 长连接**（不是 URL）
4. 确认已添加 `im.message.receive_v1`
5. **在机器人 connected 状态下点保存**

**④ 群里要 @ 机器人**

默认只有 **@梁师** 的消息才会推给机器人，纯文字不 @ 不会回。

**⑤ 运行排查脚本**

```bash
cd ~/Projects/feishu-bazi-bot
chmod +x 排查飞书事件.sh
./排查飞书事件.sh
```

按提示在 30 秒内 @ 机器人发「帮助」，看终端有没有「事件到达」。

### Q：打开了机器人，但底部没有输入框、不能打字？

**原因：** 你从「工作台 → 应用详情」进入了机器人的**介绍页**，不是**聊天会话**。

**正确打开方式（任选一种）：**

#### 方法 A：从「消息」里找机器人（推荐）

1. 打开飞书 App，点底部 **「消息」**（不是工作台）
2. 点右上角 **「✏️」** 或 **「+」** → **「发起会话」**
3. 搜索 **梁师**（你的应用名）
4. 在结果里选带 **「机器人」** 标签的那一个
5. 进入后，底部会出现 **输入框**，发：`帮助`

#### 方法 B：在群里 @ 机器人（最简单）

1. 随便建一个测试群（或现有群）
2. 群设置 → **群机器人** → **添加机器人** → 选 **梁师**
3. 在群里正常打字，**@梁师** 发：`帮助`

#### 方法 C：手机飞书

1. 底部 **消息** → 顶部搜索 **梁师**
2. 点 **联系人** 里的机器人（不是工作台里的应用图标）
3. 进入单聊后应能看到输入框

**若仍无输入框，请检查开放平台：**

| 检查项 | 位置 |
|--------|------|
| 机器人已启用 | 应用能力 → 机器人 → **开启** |
| 可用范围包含你 | 应用发布 → 可用范围 → 添加自己 |
| 权限已开通 | `im:message`、`im:message:send_as_bot` |
| 应用已发布 | 版本管理与发布 → 已上线/测试版 |

### Q：飞书页面找不到机器人？

**重要：机器人不会自动出现在聊天列表里**，需要你主动打开：

1. **先确认开放平台配置完成**（见上文第一步 1—6 步）
2. 打开飞书 App → 底部 **「工作台」**
3. 顶部 **搜索框** 输入你创建的应用名（如「梁派命理」）
4. 点击进入 → 在应用里找到 **「机器人」或「发消息」** 入口
5. 或在任意群聊 → **设置 → 群机器人 → 添加机器人** → 选你的应用

**同时**：电脑上的 `启动机器人.command` 窗口必须开着，且终端**不能**再出现 SSL 红色报错。

若终端报 `SSL: CERTIFICATE_VERIFY_FAILED`，请先运行：
```bash
cd ~/Projects/feishu-bazi-bot
./修复SSL并测试.sh
```

### Q：终端显示已连接，但飞书没反应？

1. 确认 `启动机器人.command` 窗口还开着  
2. 确认飞书事件订阅是 **WebSocket 长连接**  
3. 确认应用已发布，且你是测试用户  
4. 看终端有没有报错

### Q：提示 AI 服务不可用？

检查 `.env` 里 `LLM_API_KEY` 是否正确，DeepSeek 账户是否有余额。

### Q：必须一直开着 Mac 吗？

是的，当前方案是**在你电脑上运行**。电脑休眠或关闭，机器人会离线。  
若以后需要 24 小时在线，可再部署到云服务器（可后续再做）。

### Q：和 Cursor 里聊天一样吗？

回复风格与梁派体系一致（已内置导师提示词 + 本地排盘）。  
底层是 DeepSeek 大模型，不是 Cursor 本身，但**问法、知识体系与之前对话相同**。
